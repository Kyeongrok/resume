### MVVM 추상화·캡슐화·설계 능력은?

에이전트 개발 프로젝트에서 초기 개발 단계부터 DI(의존성 주입) 기반 아키텍처를 설계하여 고도화 단계 전환 비용을
크게 낮췄습니다. 그 결과 신규 기능 추가에 예상됐던 1개월의 개발 기간을 1주일로 단축하는 성과를 냈습니다.
확장을 고려한 설계가 실제 납기와 품질에 직결된다는 것을 몸소 경험했습니다.

### DI왜 쓰나?
DI구조로 만들어야 교체 할 수 있게 만들 수 있고 단위 테스트를 만들 수 있기 때문 이제는 습관입니다.
젖가락질 하는 것과 비슷 합니다.

### MVVM구조로 만들면 좋은 점
View의 로직을 분리 할 수 있습니다. 예를들면 '실행'버튼을 단순히 '실행버튼 클릭'이벤트를 발생시키는 도구로 만드는 것 입니다.
이벤트 리스너에 비스니스 로직이 많이 들어 있으면 나중에 그 버튼을 강제로 트리거 하는 로직을 걸어야 합니다.

### 6년된 레거시 Winform을 WPF로 전환
회사의 숙원사업이었던 Winform앱 재개발이 필요할 것 같아 진행하고 있습니다.
먼저 한 작업은 .Net Framework 4.7.2를 Dotnet8로 전환 한 것입니다.
전환 과정에서 차트 라이브러리 하나가 호환성이 안맞아서 Designer가 안되는 문제가 있었습니다.
다행히 Dotnet 상위 버전으로 포팅된 라이브러리가 있어서 교체 했습니다.

### Winform App을 WPF로 변환한 과정
1. 기존 앱에 ViewModel을 만듭니다.
2. WPF화면을 만듭니다.
3. 기존 ViewModel을 연결 합니다.
4. 기존 화면과 비교를 위해 기존 화면은 Ctrl + Click으로 만듭니다.
